The core issue is that WordPress stores site URLs in the database and will redirect or break if requests arrive at different domains. Lando configures sites like `wp-museum.lndo.site` internally, but Tailscale Serve would expose them at `your-machine.ts.net`, causing WordPress to see a mismatched URL. Solutions would involve either remapping the .ts.net domain to resolve to the internal .lndo.site address, using WordPress plugins to handle multiple URLs, or reconfiguring WordPress's siteurl/home options dynamically based on the request hostname. The conversation included a brief tangent about `newgrp docker` for managing group permissions.
