Prepare plugin for WordPress.org directory submission

Bring the plugin into compliance with WordPress.org directory guidelines
for its first public submission.

Licensing & metadata
- Relicense from MIT to GPLv2-or-later in LICENSE.md, composer.json, and
  README.md; document GPL compatibility of bundled wp-combine-queries.
- Add required plugin header fields: Plugin URI, Requires PHP, Tested up
  to, Domain Path, License, License URI.
- Add readme.txt (WordPress.org format) with description, FAQ, changelog,
  and upgrade notice.
- Add languages/wp-museum.pot translation template and Domain Path.

Security hardening
- Add defined('ABSPATH') || exit guards to all plugin PHP entrypoints.
- Escape output (esc_url/esc_attr/esc_html/tag_escape/sanitize_html_class)
  throughout block render templates; notably the collection, advanced/
  basic/embedded search, object-infobox, and collection-main-navigation
  renderers.
- Sanitize \$_POST/\$_GET in admin-side migration and meta-box save
  paths; add nonce + capability checks to the force-migration flow and
  the view-migration-log action.
- Default DB_SHOW_ERRORS to false (was true) so DB errors are no longer
  surfaced in production.

Tooling & release
- composer.json: add lint/lint:fix/format/phpcs/phpcbf/phpunit scripts.
- phpcs.xml.dist: register wpcs, phpcsutils, and phpcsextra installed
  paths so WordPress coding standard sniffs run.
- release.sh: bundle readme.txt, LICENSE.md, and languages/ into the
  release zip at the plugin root.
- Add .env.example documenting WP_HOME and related Compose variables.
- README.md: document the Docker Compose workflow as the default dev
  environment (Lando remains supported).
