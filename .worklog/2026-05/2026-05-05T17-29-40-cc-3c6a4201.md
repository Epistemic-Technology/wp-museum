Investigation revealed that Tab key was being intercepted by Gutenberg's writing-flow handler and routed to the sidebar, breaking navigation through all fields (not just rich text). Created a Playwright test that reproduces the issue by tabbing through a five-field object editor. Fixed it by adding a capture-phase Tab handler to the fields container that manually advances focus through focusable elements in DOM order, while allowing Tab at boundaries to propagate normally. The strengthened test now confirms focus advances through each field sequentially, staying within the block.

**First prompt:** There's an issue where when you're editing a museum object that has rich text fields (see @src/blocks/object-meta/edit.js ), if you're trying to edit the object with keyboard navigation, it works fine as you tab through fields until you hit a rich text field, and then the tab order gets wonky and you end up tabbing through the block options or something else in the sidebar. Obviously we want people to be able to tab through fiels in an order than makes sense. I think the first step is to see whether there is an existing playwright test that can replicate this issue or, if not, we can make one that does.
